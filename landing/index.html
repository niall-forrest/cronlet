How It Works

Create a task. Define a schedule. We run it.

Three ways to create tasks: visual dashboard, REST API, or connect AI agents via MCP.

1

Create a task

Use the dashboard, API, or SDK. Define what to run: a webhook, a multi-step workflow, or JavaScript code.

POST /v1/tasks { "name": "sync-stripe", "handler": { "type": "webhook", "url": "https://..." } }
2

Set a schedule

Pick from intervals, daily times, weekly days, or cron expressions. Add timezone support.

"schedule": { "type": "daily", "times": ["09:00", "17:00"], "timezone": "America/New_York" }
3

We handle the rest

Cronlet runs your task on time, retries on failure, and sends you the results via callback or dashboard.

// Callback payload { "event": "task.run.completed", "status": "success", "output": { ... } }
Handler Types

Three ways to handle scheduled tasks

From simple webhooks to multi-step workflows with data passing between steps.

webhook

Webhook

Call your own endpoints on schedule with auth headers

{
  "type": "webhook",
  "url": "https://api.example.com/sync",
  "auth": {
    "type": "bearer",
    "secretName": "API_TOKEN"
  }
}
tools

Tools

Chain HTTP, email, Slack steps with data passing

{
  "type": "tools",
  "steps": [
    { "tool": "http.get", ... },
    { "tool": "slack.send", ... }
  ]
}
code

Code

Run JavaScript directly on our infrastructure

{
  "type": "code",
  "runtime": "javascript",
  "code": `
    const res = await fetch(url);
    return res.json();
  `
}
For AI Agents

Give agents temporal superpowers

Connect via MCP and your AI agents can create, manage, and monitor scheduled tasks autonomously. Results flow back via callbacks, enabling self-maintaining systems.

  • Natural language schedules

    "every 15 minutes", "weekdays at 9am", "last friday of the month"

  • Callback-driven loops

    Task completes → agent receives results → agent reacts → creates new tasks

  • Approval workflows

    Critical operations require human approval before execution

  • Context preserved

    Attach metadata to tasks—intent, conversation ID—returned in every callback

Platform Features

Everything you need for reliable scheduling

Enterprise-grade infrastructure without the enterprise complexity.

Configurable Retries

Linear or exponential backoff. Set max attempts, initial delays, and let us handle transient failures.

Execution Timeouts

Per-task timeout limits prevent runaway jobs. Tasks are terminated cleanly when limits are exceeded.

Failure Alerts

Email or webhook notifications when tasks fail. Never miss a broken job again.

Encrypted Secrets

Store API tokens and credentials securely. Reference them in your tasks without exposing values.

Audit Logging

Complete audit trail of every action. Filter by actor, action type, or date range for compliance.

Team Access Control

Role-based permissions: owner, admin, member, viewer. Manage who can create, edit, or view tasks.
