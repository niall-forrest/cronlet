## What `cronlet dev` runs

`cronlet dev` starts:

1. job discovery
2. cron scheduler
3. dashboard API + UI
4. file watcher for job files

Dashboard default URL: `http://localhost:3141`.

## Startup output is your source of truth

On boot, cronlet prints:

- config file path (`Config: ...`)
- jobs source (flag, config, or auto-detect)
- resolved jobs directory
- watch directories
- discovered jobs

If behavior is unexpected, check these lines first.

## File watch behavior

Watch targets are resolved using the same logic as discovery:

1. `--dir`
2. `jobsDir` in config
3. default directories (`./jobs`, `./src/jobs`, `./app/jobs`)

After file changes, cronlet reloads discovery and reschedules jobs.

## Dashboard update model

The dashboard receives live events over Server-Sent Events (`/api/events`).

- primary path: push updates via SSE
- fallback: polling every 5 seconds if EventSource is unavailable or disconnected

This keeps local feedback fast while still working in restricted browser/runtime environments.

## Practical DX tips

- Keep job files under one clear directory (`src/jobs` is easiest in monorepos).
- Use explicit `config.name` only when you need a stable name different from file path.
- Prefer `npx cronlet validate` before committing schedule/config changes.
