## Job shape

A cronlet job is a default export created with `schedule()`:

```ts
import { schedule, daily } from "cronlet";

export default schedule(daily("09:00"), async (ctx) => {
  await sendReport(ctx.jobId);
});
```

Each job has:

- a schedule descriptor (`cron`, human-readable text, optional timezone)
- job config (retry, timeout, concurrency, hooks)
- a handler function

## Schedule builders

Use typed builders for most cases:

- `every("15m")`
- `daily("09:00")`
- `weekly("fri", "09:00")`
- `monthly(1, "09:00")`
- `monthly("last-fri", "17:00")`
- `cron("0 9 * * 1-5")`

Timezone can be set with `.withTimezone("America/New_York")`.

## Deterministic job IDs

When jobs are discovered from files (CLI and worker mode), IDs are deterministic:

1. If `config.name` is provided, it becomes the job ID.
2. Otherwise, cronlet uses the file path relative to your resolved jobs directory.

Example:

- `src/jobs/billing/sync-stripe.ts` -> `billing/sync-stripe`

If two jobs resolve to the same ID, discovery raises a conflict with both file paths.

## Job context available in handlers

```ts
interface JobContext {
  jobId: string;
  jobName: string;
  runId: string;
  scheduledAt: Date;
  startedAt: Date;
  attempt: number;
  signal: AbortSignal;
}
```

Use `signal` to support graceful cancellation in long-running work.

## Concurrency and catchup

Per-job overlap behavior:

- `concurrency: "skip"` (default): skip ticks while a run is active
- `concurrency: "allow"`: allow overlapping runs
- `concurrency: "queue"`: serialize runs and execute every tick in order

`catchup: true` only applies to `concurrency: "skip"` and queues one follow-up run when skips happen.

## Retry and timeout semantics

```ts
schedule(
  daily("09:00"),
  {
    retry: {
      attempts: 3,
      backoff: "exponential",
      initialDelay: "30s",
    },
    timeout: "5m",
  },
  async () => {
    await sendReport();
  }
);
```

Duration fields accept `ms`, `s`, `m`, `h`, `d`.
