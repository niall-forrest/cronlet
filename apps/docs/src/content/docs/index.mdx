`cronlet` is a TypeScript-first scheduling toolkit for Node.js apps. It is built for teams that want cron jobs to live with application code instead of a separate orchestration product.

## Why teams use cronlet

- Typed schedule builders: `every`, `daily`, `weekly`, `monthly`, `cron`
- File-based discovery with deterministic job IDs
- Local dashboard with live job status and manual triggering
- Built-in retries, timeouts, and overlap controls
- Deploy helpers for Vercel and a standalone worker mode for any Node host

## Where cronlet fits

Use cronlet when you want scheduling integrated into your existing app and repo:

- Next.js monorepos
- API/backoffice services with recurring tasks
- Jobs that need shared app code and types

If you need distributed workflows, multi-step orchestration, or cross-service state machines, cronlet is usually too small by itself.
